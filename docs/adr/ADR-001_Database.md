#
ADR-001: Вибір СУБД PostgreSQL 16
 
##
Статус
Accepted
 
##
Контекст
Система автоспортивного форуму "AutoSport Hub" вимагає високої надійності при збереженні пов'язаних даних (користувачі, статті, коментарі, сповіщення). Очікуване навантаження: до 10 000 активних користувачів та велика кількість одночасних запитів у пікові години (під час трансляцій гонок). Потрібна підтримка складних реляційних зв'язків між категоріями, публікаціями та ієрархією коментарів.
 
##
Рішення
Використовувати PostgreSQL 16 як основну реляційну СУБД.
 
##
Обґрунтування
1. Повна відповідність ACID: Гарантує цілісність даних при одночасному коментуванні та оновленні статусів статей редакторами.
2. Підтримка JSONB: Зручно для зберігання гнучких метаданих статей (результати гонок, характеристики болідів) без зміни схеми БД.
3. Висока продуктивність: Ефективна робота з індексами при повнотекстовому пошуку новин та вибірці стрічки за категоріями.
 
##
Альтернативи
- MySQL: Відхилено через менш гнучку роботу з JSON-типами та слабший оптимізатор складних запитів.
- MongoDB: Відхилено, оскільки система є чітко реляційною і вимагає складних JOIN-запитів для побудови стрічки форуму.
 
##
Наслідки
###
Позитивні
- Надійна робота зі складною структурою форуму.
- Широкі можливості масштабування при зростанні аудиторії.
###
Негативні
- Потрібні навички адміністрування реляційних БД.
