#
ADR-003: Технологічний стек (NestJS + React)
 
##
Статус
Accepted
 
##
Контекст
Для автоспортивного форуму критично важливою є інтерактивність: «живі» коментарі, миттєві сповіщення та плавний інтерфейс. Команда володіє мовою TypeScript. Потрібна підтримка WebSockets для реалізації live-обговорень гонок.
 
##
Рішення
Використовувати Node.js (NestJS) для Backend та React.js для Frontend (спільна мова - TypeScript).
 
##
Обґрунтування
1. Єдина мова (TypeScript): Дозволяє використовувати спільні типи даних для фронтенду та бекенду, зменшуючи кількість помилок.
2. Ефективність WebSockets: NestJS має вбудовану підтримку Socket.io, що дозволяє реалізувати оновлення коментарів у реальному часі.
3. Продуктивність: Асинхронна природа Node.js ідеально підходить для великої кількості I/O операцій та запитів до бази даних.
 
##
Альтернативи
- Python (Django): Відхилено, оскільки складніше реалізувати високонавантажені real-time функції (WebSockets) порівняно з Node.js.
- PHP (Laravel): Відхилено через застарілий підхід (server-side rendering), що ускладнює створення динамічного SPA-додатка.
 
##
Наслідки
###
Позитивні
- Сучасний інтерактивний інтерфейс для фанатів.
- Висока швидкість розробки завдяки спільній екосистемі npm.
###
Негативні
- Екосистема JavaScript швидко змінюється, що вимагатиме регулярного оновлення залежностей.
